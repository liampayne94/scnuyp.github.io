AJAX

发表于 2017-05-27 | 阅读次数

AJAX全称是Asynchronous Javascript And XML，最大的作用在于能够不用重新载入页面就能够更新部分页面的内容，下面介绍其三种实现方式：

原生实现

前端的页面如下：

首先，要获得一个Xmlhttp对象，以便于之后打开连接，值得注意的是，IE浏览器和非IE浏览器（谷歌浏览器等）获取xmlhttp对象的方式是不一样的。

在原生的方法里面，我们可以调用open方法通过“get”或者“post”的方法打开连接

P.S：与 POST 相比，GET 更简单也更快，并且在大部分情况下都能用。

然而，在以下情况中，请使用 POST 请求：

  • 无法使用缓存文件（更新服务器上的文件或数据库）
  • 向服务器发送大量数据（POST 没有数据量限制）
  • 发送包含未知字符的用户输入时，POST 比 GET 更稳定也更可靠

而当readystate发生改变的时候就可以调用一个叫complete的回调函数

在回调函数中，我们可以写自己想要实现的方法

PS：用post方法的话，一定要加这句

xmlHttp.setRequestHeader("Content-Type", "application/x-www-form-urlencoded");

jQuery实现

jQuery的实现方法与原生的方法大同小异，主要区别在于：jQuery将方法封装了起来，我们只需要调用jQuery中的get或者post方法来实现。

另外要说明一下$.post(url,{param:”param”},function(data){})中的返回对象data

jQuery文档中说,这里的data类型可以是：xml, html, script, json, text, _default

DWR

由上可以看出，

1、在原生的方法中XMlhttp控件存在兼容性的问题（jQuery可以解决这个问题）

2、当返回的数据类型为xml或者json的时候，还需要用工具类进行处理，相对来说很麻烦

3、页面接受并处理服务端返回的结果的时候，有时会比较麻烦（jQuery可以解决）

针对以上问题，出现了DWR，可以理解为DWR帮开发人员省去了一些中间步骤，使之专注于通过js调用服务端的方法

DWR核心就是一个Servlet,我们需要在web.xml中按照配置servlet的方法进行配置，并在统一目录下配置dwr.xml
