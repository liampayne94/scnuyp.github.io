Hadoop中的MapRedure

发表于 2017-08-04 | 阅读次数

前言

MapReduce是hadoop的计算框架，要理解hadoop的实现原理，理解HDFS和MapReduce是必不可少的。参考《Hadoop权威指南》和博客上的一些资料，写写自己的见解。

什么是Hadoop

Apache Hadoop是一款支持数据密集型分布式应用并以Apache 2.0许可协议发布的开源软件框架。它支持在商品硬件构建的大型集群上运行的应用程序。Hadoop是根据Google公司发表的MapReduce和Google档案系统的论文自行实作而成。

Hadoop框架透明地为应用提供可靠性和数据移动。它实现了名为MapReduce的编程范式：应用程序被分割成许多小部分，而每个部分都能在集群中的任意节点上执行或重新执行。此外，Hadoop还提供了分布式文件系统，用以存储所有计算节点的数据，这为整个集群带来了非常高的带宽。MapReduce和分布式文件系统的设计，使得整个框架能够自动处理节点故障。它使应用程序与成千上万的独立计算的电脑和PB级的数据。　　

MapReduce的工作过程

MapReduce的工作过程 主要分为两个阶段：Map阶段和Reduce阶段，每个阶段都由键值对的方式处理数据，亦即是框架把作业的输入看为是一组 键值对，同样也产出一组 键值对做为作业的输出，从（http://weixiaolu.iteye.com/blog/1474172）拿来一张很好的图片：

分析

1、在客户端启动一个程序

2、向JobTracker请求一个作业号

3、将运行所需要的资源复制一份到HDFS中，包括程序打包而成的Jar包，配置文件以及客户端计算得到的划分信息（这个任务应该启动多少个map任务）。这些文件都存放在JobTracker专门为该作业创建的文件夹中。文件夹名为该作业的Job ID

4、JobTracker接收到作业之后，把它放在一个消息队列之中，等待作业调度器对其进行调度，同时向HDFS获得作业划分的信息，

5、当作业调度器根据调度算法调度到该作业的时候，会创建一个Map任务，交由TaskTracker执行。对于map和reduce任务，TaskTracker根据主机核的数量和内存的大小有固定数量的map槽和reduce槽。

7、TaskTracker会定期向JobTracker发送一个心跳包以保持连接，同时心跳中还携带着很多的信息，比如当前map任务完成的进度等信息。当JobTracker收到作业的最后一个任务完成信息时，便把该作业设置成“成功”。当JobClient查询状态时，它将得知任务已完成，便显示一条消息给用户。

8910、从HDFS中获取得到作业的资源文件，并且登录子JVM启动任务
